# Alert System User Guide

> [← Back to Documentation Index](README.md)

## Overview

The Alert System provides automated notifications for important portfolio events, helping you stay informed about significant changes without constantly monitoring the dashboard.

## Features

- **Multiple Alert Types**: Drawdown, price changes, position changes, volume spikes, rebalancing, benchmark divergence
- **Flexible Configuration**: Per-run or global alerts with customizable thresholds
- **Multiple Channels**: Email, SMS (placeholder), and in-app notifications
- **Rate Limiting**: Minimum interval controls prevent alert spam
- **Alert History**: Complete history of all sent alerts

## Accessing Alert Management

1. Launch the dashboard: `streamlit run src/app/dashboard/app.py`
2. Navigate to **Utilities** section in the sidebar
3. Click **🚨 Alert Management**

## Creating Alert Configurations

### Step 1: Navigate to Create Alert Tab

1. Go to **🚨 Alert Management** page
2. Click the **➕ Create Alert** tab

### Step 2: Configure Alert

1. **Select Run**: Choose a specific run to monitor, or select "All Runs (Global)" for global alerts
2. **Alert Type**: Choose from:
   - **Drawdown**: Portfolio drawdown alerts
   - **Price Change**: Individual stock price movement alerts
   - **Position Change**: Large position weight changes
   - **Volume Spike**: Unusual trading volume
   - **Rebalancing**: Rebalancing reminders
   - **Benchmark Divergence**: Portfolio underperformance vs benchmark

3. **Threshold**: Set the alert threshold
   - For drawdown: Negative value (e.g., -0.10 for 10% drawdown)
   - For price change: Positive value (e.g., 0.05 for 5% change)

4. **Notification Channels**: Select one or more:
   - **In-App**: Alerts stored in dashboard (always available)
   - **Email**: Requires SMTP configuration (see Configuration section)
   - **SMS**: Placeholder for future integration

5. **Email/Phone**: If email or SMS selected, provide contact information

6. **Minimum Interval**: Set minimum hours between alerts (prevents spam)

7. Click **Create Alert Configuration**

## Alert Types Explained

### Drawdown Alerts
- **Purpose**: Warn when portfolio drawdown exceeds threshold
- **Threshold**: Negative percentage (e.g., -0.10 = 10% drawdown)
- **When Triggered**: Current drawdown ≤ threshold

### Price Change Alerts
- **Purpose**: Notify of significant stock price movements
- **Threshold**: Positive percentage (e.g., 0.05 = 5% change)
- **When Triggered**: |Price change| ≥ threshold

### Position Change Alerts
- **Purpose**: Alert on large position weight changes
- **Threshold**: Percentage change (e.g., 0.05 = 5% weight change)
- **When Triggered**: Position weight change ≥ threshold

### Volume Spike Alerts
- **Purpose**: Detect unusual trading volume
- **Threshold**: Multiplier (e.g., 2.0 = 2x average volume)
- **When Triggered**: Volume ≥ threshold × average volume

### Rebalancing Alerts
- **Purpose**: Remind when rebalancing may be needed
- **Threshold**: Portfolio drift percentage
- **When Triggered**: Portfolio drift ≥ threshold

### Benchmark Divergence Alerts
- **Purpose**: Warn of portfolio underperformance
- **Threshold**: Underperformance percentage
- **When Triggered**: Portfolio return - Benchmark return ≤ -threshold

## Configuration

### Email Configuration

To enable email notifications, configure SMTP settings in your environment or config:

```python
smtp_config = {
    'host': 'smtp.gmail.com',
    'port': 587,
    'username': 'your-email@gmail.com',
    'password': 'your-app-password',
    'use_tls': True,
    'from': 'your-email@gmail.com'
}

alert_service = AlertService(smtp_config=smtp_config)
```

**Note**: For Gmail, you'll need to use an [App Password](https://support.google.com/accounts/answer/185833) instead of your regular password.

## Best Practices

1. **Start Conservative**: Begin with higher thresholds and lower frequency
2. **Use Minimum Intervals**: Prevent alert fatigue with appropriate intervals
3. **Monitor Alert History**: Review what alerts are actually useful
4. **Adjust Thresholds**: Fine-tune based on your risk tolerance
5. **Use Multiple Channels**: Combine in-app and email for important alerts

## Troubleshooting

### Alerts Not Sending

1. **Check Configuration**: Verify alert is enabled and threshold is appropriate
2. **Check SMTP**: If using email, verify SMTP configuration
3. **Check Minimum Interval**: Alerts may be throttled by minimum interval
4. **Check Alert History**: View history to see if alerts were generated but not delivered

### Too Many Alerts

1. **Increase Threshold**: Make threshold more restrictive
2. **Increase Minimum Interval**: Reduce alert frequency
3. **Disable Unnecessary Alerts**: Remove alerts you don't need

### Email Not Working

1. **Verify SMTP Settings**: Check host, port, username, password
2. **Check Firewall**: Ensure SMTP port is not blocked
3. **Use App Password**: For Gmail, use App Password instead of regular password
4. **Check Spam Folder**: Alerts may be filtered as spam

## API Usage

### Programmatic Alert Creation

```python
from src.analytics.alert_system import AlertService, AlertType, AlertChannel

alert_service = AlertService()

# Create alert
config = alert_service.create_alert_config(
    alert_type=AlertType.DRAWDOWN.value,
    run_id='your_run_id',
    user_email='user@example.com',
    threshold=-0.10,
    channels=[AlertChannel.EMAIL.value, AlertChannel.IN_APP.value],
    min_interval_hours=1.0
)

# Check and send alerts
alert_data = {
    'type': AlertType.DRAWDOWN.value,
    'level': 'critical',
    'message': 'Portfolio drawdown: -12.5%',
    'value': -0.125
}
sent_alerts = alert_service.check_and_send_alerts('your_run_id', alert_data)
```
